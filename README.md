# APOLLO Poster Reproduction


1. `The Dilemma`: AdamW is memory-bound, GaLore is SVD-bound, SGD is fast but weak for Transformers.
2. `AdamW-level Performance`: compare convergence and final validation quality on a small GPT-2 setup.
3. `SGD-like Memory & Speed`: profile memory, max batch size, throughput, and optimizer step time.
4. `Sanity Checks`: scaling-factor ratios and directional sharpness.
5. `Ablation & Failure Modes`: rank sensitivity and why `r = 1` needs APOLLO-Mini.

## Smarter Experiment Split


- `full_pretrain`:
  - model: GPT-2 124M
  - data: WikiText-2
  - optimizers: SGD, AdamW, GaLore rank 1/4, GaLore rank 1/8, APOLLO rank 1/4, APOLLO rank 1/8, APOLLO-Mini
  - outputs reused for:
    - convergence race
    - table-2-like comparison
    - Pareto frontier
    - scaling-factor traces

- `microbench`:
  - representative optimizers only
  - outputs reused for:
    - memory breakdown
    - max batch size
    - throughput
    - optimizer-step spikes
    - table-7-like summary

- `ablation`:
  - GaLore rank sensitivity
  - APOLLO rank sensitivity
  - APOLLO channel-wise `r=1` vs APOLLO-Mini tensor-wise `r=1`

- `sharpness`:
  - post-hoc analysis using checkpoints or early-step snapshots from selected runs

## Main Commands

```bash
pip install -e .
python scripts/run_suite.py --suite configs/suites/full_pretrain.yaml --output-root results
python scripts/run_suite.py --suite configs/suites/microbench.yaml --output-root results
python scripts/run_suite.py --suite configs/suites/ablation.yaml --output-root results
python scripts/analyze_sharpness.py --suite-dir results/full_pretrain
python scripts/make_plots.py --results-root results
python scripts/build_poster_assets.py --results-root results
```

To resume an interrupted training suite:

```bash
python scripts/run_suite.py --suite configs/suites/full_pretrain.yaml --output-root results --resume latest
```

To resume a single experiment from a specific checkpoint:

```bash
python scripts/train_experiment.py --config configs/base.yaml --experiment-name adamw --output-root results/full_pretrain --resume results/full_pretrain/adamw/checkpoint_step_1000.pt
```

## Notes

- The training loop is nanoGPT-style, but fully instrumented for memory and timing.
- GaLore is implemented as an SVD-based low-rank gradient optimizer to preserve the expected periodic latency spikes.
- APOLLO logs structured scaling traces at selected steps so you can reproduce the paper-style scaling-factor plot.
- Pressure-test style plots are generated by benchmarking under a configurable memory budget, not by changing the physical GPU.
